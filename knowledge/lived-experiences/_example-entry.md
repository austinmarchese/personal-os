# 2024-03-15 — Learned to scope projects smaller

## What Happened

Started building a "simple" habit tracker app. Three weeks later, I had a database schema for 47 features and zero working code. Realized I was designing for every possible use case instead of building the minimum thing that works.

Scrapped it. Started over. Built just the core loop in one weekend: log a habit, see a streak. That's it. Actually shipped it.

## Key Learning

The first version should be embarrassingly simple. If you're not embarrassed by v1, you waited too long to ship. Features come from usage, not imagination.

## Quotable

"I spent three weeks designing and zero weeks building. Now I build the dumbest possible version first and let real usage tell me what's missing."
